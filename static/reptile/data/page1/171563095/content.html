信息化软件项目 1) 支持多种平台系统，如Windows, Linux操作系统； 2) ▲支持热备高可用性部署，主备机之间配置、消息库可实时同步，当主机发生故障时，备机可在不需人工干预的情况下自动启动，消息在备机中继续运行，当主机修复后，消息会转回主机中继续处理； 3) 支持HL7v2、HL7v3、国家互联互通CDA标准、FHIR、XML、JSON等标准和规范，提供对这些标准处理的工具； 4) 内置HL7v2、HL7v3等标准库； 5) 支持多种通讯协议，包括但不限于TCP/IP、SOAP Web 服务、REST Web 服务、文件、定时器、DLL、Kafka、数据库等； 6) 内嵌简单代码映射lookup表配置功能、H2内存数据库； 7) 支持通用的JAVA脚本开发功能，包括 Groovy脚本, 支持对JSON, XML 结构数据的脚本处理； 8) 针对同一服务可同时支持接口模式和集成模式； 1.1.2 用户界面要求 1) ▲开发界面应均为网页界面；网页界面支持中英文一键即时切换； 2) 能在同一个界面中完成流程开发、服务监控等工作，并能显示异常错误队列。 3) ▲拖拉式图形化路由设计，并支持路由间的衔接和串联； 4) ▲能提供全局视图显示整个流程完整流通线路，用户能直观查看包含多终端，多路由的完整消息处理流程，在一个视图页面上能看到整体业务流程图； 5) 提供便捷的图形化数据映射配置界面，并能支持通过代码编写进行数据映射配置。 （1）服务内容 ①为保证新院12月份准时开诊，需要将医院现有信息系统及数据无缝平移到新院区，并要求保证无缝平移后的信息系统和数据完整无缺，保证医院及时开诊。 ②同时为了应对医院发展需求需对原有老院区的部分业务应用系统进行升级扩充。 ③升级扩充涉及的医院信息系统包括：集成平台、LIS系统扩充、手术麻醉系统扩充、重症监控管理系统扩充、医院智能审方系统、患者主索引、临床诊疗知识库系统、高值耗材管理系统、急诊系统、病理系统、医学证明及报告管理系统、移动设备管理、病案质控、心电系统扩充、PACS系统扩充、临床数据中心、患者临床视图、BI系统扩充、合理用药、HIS系统、电子病历系统。 （2）维保时间 ①集成平台、LIS系统扩充、手术麻醉系统扩充、重症监控管理系统扩充、医院智能审方系统、患者主索引、临床诊疗知识库系统、高值耗材管理系统、急诊系统、病理系统、医学证明及报告管理系统、移动设备管理、病案质控、心电系统扩充、PACS系统扩充、临床数据中心、患者临床视图、BI系统扩充提供2年免费服务。 ②集成平台提供3年接口免费服务，对接系统需满足集成平台类接口规范，不包含银医、药商物流、互联网医院、区域互联等需配合原系统流程改造类的接口应用。 ③合理用药、HIS系统、电子病历系统提供3年免费服务。 ④要求提供3年2人驻场服务。 包 1.0 690 690
