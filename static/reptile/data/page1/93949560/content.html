天津市人民政府国有资产监督管理委员会机关 天津市人民政府国有资产监督管理委员会采购门户网站项目 (项目编号:FZK2018-3-038)中标公告

天津市人民政府国有资产监督管理委员会采购门户网站项目 （一）业务需求依据国办发57号文《关于加强政府网站信息内容建设的意见》、国办发〔2015〕15号《国务院办公厅关于开展第一次全国政府网站普查的通知》、国办发〔2017〕47号《国务院办公厅关于印发政府网站发展指引的通知》提出的推进“集约化建设”的要求，将网站集群建设成具有创新影响力的“互联网+政务”政府门户集群网站。（二）功能需求建成“内容管理系统、数据中心管理系统、绩效管理系统、信息公开系统、在线访谈系统、便民咨询系统、网上信访系统、问卷调查系统、投票系统、横幅管理系统、微信管理系统、政务微博系统、在线办事系统、在线表单系统、数据采集系统、全文检索系统、移动设备自适应支撑系统”等应用系统。（三）数据需求整个网站群支持“统一的数据中心管理”，要求实现对整个网站群间的数据管理、统计、共享和交换。（四）性能需求网站群平台应当满足以下性能指标要求： （1）在网络稳定的环境下操作性界面单一操作的系统响应时间＜2秒。（2）正常500个并发请求的性能要求。 （3）支持年数据量为300万条记录数、500GB的数据增量 （4）系统提供7×24小时的连续运行，平均年故障时间＜2天，平均故障修复时间＜3小时。（五）安全需求对于政府网站群项目来说，安全性是首要考虑的问题。应在网络安全各个环节采用有力措施，保证系统网络整体安全性，系统能够动态监控网站安全，网站能在遭受到非法攻击后迅速恢复。（1）平台安全配置支持自定义后台管理目录。支持设定管理员身份验证票过期时间。支持控制是否启用后台管理认证码。支持控制是否启用后台来访页验证。支持控制是否对数据库连接字符串加密。支持控制是否启用小写URL模式。支持自定义管理员密码哈希值。（2）网页安全性 1. 前后台数据的交互正常，注册、登录、提交、修改等表单功能正确，招标单位合同内需求的功能使用正常。 2. 模板文件有存放和备份机制，在紧急情况下可以及时修复或恢复。 3. 模板与数据库相分离，标签编写规范、技术成熟，提升查询效率，使用多种技术手段防止注入攻击，确保查询数据库、前台数据呈现的安全性。 4. 对于访问者通过前台页面提交的任何数据，都要进行恶意代码检查。 5. 网站模板和数据必须有安全备份和恢复机制，确保灾难时能在很短的时间内及时恢复，确保网站的正常运行。 6. 网站底层程序的错误信息必须经过处理后再输出，防止将技术细节暴露给普通访问者。 7. 通过前台页面注册、登录的密码或其它机密数据必须用成熟加密技术加密后才能写入Cookie，密码或其它机密数据必须用成熟加密技术加密后才能通过页面的表单进行传递。（3）标签安全性 1. 标签中的HTML代码必须遵循和使用W3C标准规范，使用各种技术使页面兼容浏览器解析，使用跨平台的标准XML，可兼容各种平台。 2. 标签技术规范、易用，以合理的参数控制减少标签的数量，一种标签可以适用多种前台页面应用效果，用参数控制表达所需效果便于扩展维护。 3. 合理地运用技术手段，使标签数据查询、结构生成、样式呈现等各个环节紧密结合，提高标签整体的执行速度，高效实现特定的页面展示效果。 4. 在标签中使用防止SQL注入、过滤非法字符、连接字符串加密、防止远程访问和下载等多种技术手段，提高标签的安全。 详见招标文件 套 1.0 76.7 76.7
